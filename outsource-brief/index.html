01 · 项目定位

范围边界
一张表说清楚

合同范围三层分级：Level 1 必须包含，Level 2 明确后才算，Level 3 一律不在当前合同内。供应商不得将自身计划内容默认纳入报价或验收。

✅
Level 1 · 必须包含
纳入当前合同，不可省略
  • 底盘 CAN 指令执行与状态反馈
  • 上位机运动控制接口（速度 / 转向 / 停车）
  • 遥控器接管状态读取与反馈
  • 急停 / 软件停车链路
  • 基础 IO 状态采集
  • 心跳与通信超时检测
  • 故障码上报
  • ROS2 导航栈（定位 / 路线 / 感知 / 决策）
  • Teach & Repeat 路线录制与回放
  • 基础障碍安全停车
  • 与导航/嵌入式系统状态机联调
🔶
Level 2 · 可纳入合同
明确后生效，不默认包含
  • 成品 CAN 电机控制器选型与型号冻结
  • 除雪电机遥控 / 上位机启停控制
  • 除雪机构方向 / 速度 / 状态读取（视控制器能力）
  • 过流 / 过温 / 堵转故障状态读取
  • 除雪与底盘运动安全互锁
  • 局部避障增强能力（须另行约定）
  • 超过最低障碍停车的 Beta 增强能力
🚫
Level 3 · 默认不含
须专项 SOW，不得混入
  • 自研底盘控制器 / 电机驱动板硬件
  • 功率电路设计 / PCB 设计
  • BMS 硬件开发
  • 整机功能安全认证 / 法规认证
  • 量产级 EMC / 雪地可靠性认证
  • Gamma / Delta / Fleet / Service 能力
  • 无人值守商业运营责任
  • 整机 Engineer of Record 责任
02 · 阶段定义

Alpha 与 Beta
的本质区别

两个阶段是递进关系，不是并行。Alpha 证明机器能被安全控制，Beta 证明机器能按人定义的任务干活。

Alpha 阶段 · Prototype / Assist
人控制机器
证明机器可控 · 能安全跑起来
  • 上电 / 启动 / 可调试
  • 工控机 + 底盘控制器 + 传感器 + 通信链路协同
  • 上位机或遥控器控制前进 / 后退 / 转向 / 停车
  • 基础状态反馈（底盘 / 传感器 / 定位 / 故障）
  • 急停、人工接管、基础安全停车
  • 异常优先进入安全状态
  • 基础日志记录可复盘
  • 代码 / 配置 / 接口 / 部署方式可交付可接管
Alpha 不代表
  • 自主除雪或自动规划路线
  • 无人值守运行
  • 完整避障能力
  • 最终安全认证或法规认证
Beta 阶段 · Task / Product Demo
人定义任务
机器执行任务 · 基础任务可闭环
  • 固定场地路线录制 / 保存 / 加载 / 重复执行
  • RTK / IMU / Odometry 位置与姿态解析
  • 低速路径跟踪（≤ 0.8 m/s，横向误差 ≤ 0.25m）
  • 基础障碍检测 → 优先安全停车
  • 异常降级（定位 / 传感器 / CAN / 上位机异常停车）
  • 人工随时接管优先
  • 日志 / 测试记录 / 问题记录可复盘
  • 可用于工程演示 / 融资展示
Beta 不代表
  • 任意场地自主运行
  • 无人值守商业运营
  • 全区域覆盖路径规划
  • 量产交付或客户正式运营

04 · 交付规则

什么算交付
什么不算

每阶段正式交付必须形成可追踪、可复现、可接管的 Release Package，进入 ARCBOS 指定 GitHub 仓库。未进入 GitHub 的任何内容，不视为正式交付。

✅ 有效交付清单
  • 完整源码（进入 GitHub，可追踪 commit）
  • 配置文件 / 参数文件 / 传感器配置
  • 启动脚本 / 部署说明 / 运行验证脚本
  • 接口文档 / CAN 协议 / 路线文件格式 / 日志格式
  • 测试脚本 + 测试数据 + 测试报告
  • 故障码表 + 安全逻辑说明
  • 接线图 / IO 点表 / 控制器型号 + BOM
  • 已知问题清单（每次 Release 必附）
  • README + 交接记录
  • SBOM / 依赖许可清单（B5 / M5）
🚫 无效交付（单独使用）
  • 只交视频演示，无源码无测试报告
  • 只交二进制固件，无源码
  • 只交口头说明，无文档
  • 只交截图 / 照片
  • 只在乙方电脑 / 账号 / 远程环境可运行
  • 成品控制器但不交参数 / 接线 / 协议
  • 演示结果但无测试报告和问题清单
  • 未进入 GitHub 的任何内容
  • 命名含 final / latest / 最终版 / 改改改
  • 使用个人仓库作为主交付
安全优先级 · 不可逆转的 6 级层级
1
物理急停
最高优先级，任何自动控制不得覆盖 · 急停触发：立即禁止自动运动与除雪输出 · 安全状态恢复必须人工确认，不得自动恢复
2
遥控器 / 人工接管
人工控制优先于自动控制 · 接管时自动控制立即释放 · 通信恢复后不得自动切回自动模式
3
嵌入式底层保护
可拒绝危险指令 · IPC 崩溃或断链时 MCU 看门狗必须自主安全停车（fail-safe 实现在 MCU 侧）
4
软件安全停车
上位机 / 导航系统可请求停车 · 安全状态恢复须经人工确认
5
自动导航控制
只在安全条件满足时执行 · 不确定时默认停车（不选"降级运行"）
6
除雪作业控制
不得高于运动安全和急停 · 除雪故障须停止作业，必要时停止底盘

05 · 铁律

不可触碰的
底线原则

来自 4 份文件的最终原则，违反任一条将影响里程碑验收和付款节点。

🔀
阶段边界不可突破
当前只做 Alpha 补齐 + Beta 闭环。不能将 Gamma / Delta / Fleet / Service 混入合同。供应商计划中的合理内容可吸收，但不能扩大阶段边界或默认纳入报价。
📦
不进 GitHub 不算交付
未提交 → 不视为交付。无 README → 不视为可接管。无部署说明 → 不视为可复现。无测试记录 → 不视为可验收。无接口文档 → 不视为可联调。
⚡
接口必须以数据契约冻结
M1 必须冻结：ROS2 .msg、CAN DBC、route schema、fault_code_table、health schema，存于 contracts/ 顶层目录。名称无 Schema 不视为接口冻结。
📏
验收必须可量化
不接受无指标的"低速 / 跟踪 / 降级"。所有验收阈值在 M1 测试方案中冻结。"降级或停车"二选一时，默认取停车（面向无人值守终态）。
🔒
知识产权必须披露
背景技术在附件三填写披露表。第三方组件须披露许可证与商业化影响。未披露的背景技术和限制，不得作为限制甲方使用、修改、商业化的理由。
🌐
禁用中国镜像与服务
不得使用任何中国镜像 / 字体 / 库 / API / 云服务。代码与注释仅限英文。CI Gate 机器强制扫描，不依赖人工 review，违者 PR 即失败。
📅
每周必须正式提交
源码 / 配置 / 文档至少一类有效更新，commit message 说明变更。不接受长期本地开发后一次性打包提交，不接受口头进度代替 GitHub 提交。
🛡️
单一 Safety Arbiter
安全优先级必须由单一 Safety Arbiter 模块强制执行，不得分散在多个模块各自判断。M1 必须命名 Safety Arbiter 并定义其执行机制。
CI 强制校验 Gate
  • 检出非 ASCII 字符（源码 / 注释 / 路径）→ PR 即失败
  • 检出写死的 token / IP / 个人路径 → PR 即失败
  • 检出被禁镜像源（含任何中国 pip/npm/apt 镜像）→ PR 即失败
  • B5 Final：main 分支 CI 扫描通过（非 ASCII + 密钥）→ 方可进入 ACCEPTED_BASELINE
