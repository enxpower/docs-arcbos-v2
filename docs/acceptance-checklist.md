# Acceptance Checklist

## General

- Project context files exist.
- Scope is clear.
- Repository purpose is documented.
- No unrelated files were changed.
- No secrets are committed.
- Documentation is concise and actionable.
- All generated repository content is English-only.

## For Public HTML / Website Repositories

- Mobile layout has no horizontal scroll.
- Desktop, tablet, and mobile layouts are checked.
- Page title and description are relevant.
- Social preview metadata exists.
- Favicon exists.
- Preview image exists or is explicitly listed as missing.
- Correct VI is applied.
- No dark scheme is used unless approved.
- No external tracking is added unless approved.
- Public pages do not expose internal credentials or private logic.

## Repository-Specific Checks

- `CNAME` still points to `ref.arcbos.com` unless a domain change is explicitly approved.
- Actual GitHub Pages source is verified before deployment changes.
- Full repository tree is inspected before structural changes.
- Public ARCBOS content uses ARCBOS VI.
- Public ARCBOS presentation remains practical, technical, industrial, and execution-focused.
- No private ARCBOS engineering, supplier, investor, or credential information is exposed.
- No framework, dependency, or build system is added without approval.
- Public metadata, favicon, and preview image are added if public HTML is introduced.
