4. Alpha / Beta 能力定义

4.1 Alpha 嵌入式能力

Alpha 阶段应实现：

├─ 控制器 / MCU / 成品控制器可上电运行
├─ 底盘可执行前进 / 后退 / 转向 / 停车
├─ 上位机可通过 CAN 或约定协议控制底盘
├─ 底盘状态可反馈给上位机
├─ 遥控器可接管
├─ 急停可触发安全停车
├─ 软件停车可触发安全停车
├─ 通信丢失时进入安全状态
├─ 基础 IO 可读取
├─ 基础故障码可上报
├─ 除雪机构电控方案冻结或成品控制器联调计划冻结
└─ 代码 / 配置 / 接口 / 参数可交付

4.2 Beta 嵌入式能力

Beta 阶段应实现：

├─ 稳定执行导航系统下发的速度 / 转向 / 停车指令
├─ 提供底盘反馈、执行状态、通信健康状态
├─ 支持路径跟踪所需的运动执行稳定性
├─ 支持障碍停车、降级停车、异常停车
├─ 支持人工接管优先
├─ 支持急停最高优先级
├─ 除雪机构启停状态可接入任务状态机
├─ 除雪机构可由遥控器或上位机按约定方式控制
├─ 底层故障码可记录并上报
├─ 控制器配置、参数、固件/代码、接口文档可交付
└─ 支持 Beta 固定场地演示和工程复盘

7. 安全优先级

优先级来源原则
1物理急停最高优先级，任何自动控制不得覆盖
2遥控器 / 人工接管人工控制优先于自动控制
3嵌入式底层保护可拒绝危险指令
4软件安全停车上位机/导航系统可请求停车
5自动导航控制只在安全条件满足时执行
6除雪作业控制不得高于运动安全和急停

安全规则：

├─ 急停触发：立即禁止自动运动与除雪作业输出
├─ 遥控接管：自动控制释放，人工优先
├─ 通信超时：底盘进入安全停车或保持安全状态
├─ CAN异常：停车或进入故障状态
├─ 控制器故障：停止运动并上报故障码
├─ 除雪机构故障：停止作业机构，必要时停止底盘
├─ 定位/感知异常由导航判定，嵌入式执行停车请求
└─ 安全状态恢复必须经人工确认，不得自动恢复作业

8. 接口与数据契约

8.1 控制指令

指令来源目标必须字段备注
HEARTBEAT上位机/嵌入式双向系统健康timestamp, source, status超时触发安全策略
MODE_CMD上位机/遥控嵌入式控制器manual/auto/safe/fault模式切换需记录
MOTION_CMD导航上位机底盘执行层speed, steering/angular, enable, timestamp需限幅与安全校验
STOP_CMD上位机/安全模块底盘执行层stop_type, reason, timestampsoft_stop / emergency_request
WORK_UNIT_CMD上位机/遥控除雪机构控制层start/stop, direction, speed_if_availablespeed字段视控制器能力
RESET_CMD人工确认嵌入式控制器target, reason, operator急停后不得自动reset

8.2 状态反馈

状态方向必须字段备注
CHASSIS_FEEDBACK嵌入式 → 上位机speed_actual, steering_actual, motion_state可由底盘控制器提供
MODE_STATUS嵌入式 → 上位机manual/auto/safe/fault与状态机一致
ESTOP_STATUS嵌入式 → 上位机active, source, timestamp急停最高优先级
MANUAL_TAKEOVER_STATUS嵌入式 → 上位机active, source, timestamp人工优先
WORK_UNIT_STATUS嵌入式 → 上位机running, direction, speed_if_available, fault控制器能力不足需说明
POWER_STATUS嵌入式 → 上位机voltage, current_if_available, bms_state_if_available如BMS接口可用
FAULT_CODE嵌入式 → 上位机code, severity, module, description, timestamp必须可复盘
SYSTEM_HEALTH嵌入式 → 上位机controller, can, io, work_unit, chassis用于任务状态机

8.3 CAN / 协议交付

必须交付：

├─ CAN ID 表
├─ 报文字段定义
├─ 字节序 / 数据类型 / 缩放比例
├─ 发送周期
├─ 超时阈值
├─ 默认安全状态
├─ 故障码定义
├─ DBC 文件（如适用）
├─ 示例报文

10. 故障码体系

故障码至少覆盖：

├─ 通信故障：CAN timeout / heartbeat lost / invalid command
├─ 底盘故障：drive fault / motor feedback lost / command rejected
├─ 安全故障：estop active / manual takeover active / reset required
├─ 除雪机构故障：controller fault / overcurrent / overtemperature / stall / status unknown
└─ 电源故障：low voltage / bms alarm / power abnormal（如接口可用）

每个故障码应包含：

code / severity / module / trigger condition / action / recovery method / log requirement
