1. 文件定位

本附件用于统一 SnowBot Alpha/Beta 阶段的正式交付物、背景技术披露和第三方组件披露要求。

本附件与以下文件共同使用：

├─ 附件一-A：《SnowBot Alpha/Beta 导航与自主系统技术基线 v2.1》
├─ 附件一-B：《SnowBot Alpha/Beta 嵌入式与电控系统技术基线 v2.1》
├─ 附件二：《SnowBot Alpha/Beta System Milestones 整机系统里程碑纲领》
└─ 主合同：《PUB-2605-0039-AGR》

本附件不扩大附件一-A、附件一-B、附件二或主合同约定的开发范围；仅用于明确纳入合同范围内成果的交付形式、披露要求和验收资料。

---

2. Release Package 交付总原则

乙方每一阶段正式交付均应形成可追踪、可复现、可接管的 Release Package。

Release Package 至少应包含：

Release Package
├─ Source Code
├─ Configuration
├─ Protocol
├─ IO List
├─ Wiring Diagram
├─ Test Data
├─ Test Report
├─ Known Issues
├─ Deployment / Build / Run Instructions
├─ Version / Commit / Release Record
└─ Handover Notes

未进入甲方指定 GitHub 仓库、双方确认的正式工程交付包，或未形成可追踪版本记录的内容，不视为正式交付。

适用于导航、嵌入式、电控联调和系统级交付。

General Delivery Items

禁止作为唯一交付：

Invalid Delivery Alone

├─ 只交成品控制器但不交参数、接线和协议

6. System Milestones 对齐交付

本附件交付物应按附件二整机系统里程碑组织，不得自建脱离整机 Gate 的交付节奏。

整机 Gate交付重点Release Package 要求
A1 系统架构冻结架构、接口、控制器、IO、协议初版架构图、接口清单、CAN协议草案、IO点表、控制器选型
A2 底盘与驱动系统打通底盘运动执行与反馈底盘控制代码/配置、CAN日志、运行测试报告
A3 除雪作业模块完成除雪机构成品控制器联调控制器型号、接线图、参数配置、启停测试、联调报告
A4 Alpha 工程原型机完成Alpha 联合运行Alpha Test Report、Known Issues、基础运行日志
B2 定位与路线系统完成Teach & Repeat 基础能力路线录制/保存/加载/回放资料、测试数据、路线回放报告
B3 任务执行闭环完成路线执行、任务状态机、安全停车联合测试报告、障碍安全停车记录、异常停车记录
B5 System Demonstration & Engineering Handover系统演示能力与工程交付完整 Release Package、交接记录、培训记录、最终 Known Issues

说明：B1 产品化结构重构、B4 Beta 展示状态完成属于整机协同 Gate，不应被解释为单一导航或嵌入式供应商的独立交付责任，除非合同另行明确。

7. GitHub / 工程资产交付要求

正式交付应进入甲方指定 GitHub 仓库或双方确认的正式工程交付包。

建议仓库结构：

snowbot-autonomy
├─ Navigation / Autonomy / IPC / ROS2 / Route / Safety / Task Manager
└─ Release / Test / Docs

snowbot-embedded
├─ Embedded Control / CAN / IO / Motor Control Interface / Firmware or Controller Configuration
└─ Release / Test / Docs

最低要求：

├─ Source Code 或 Configuration 必须可追踪
├─ Protocol 必须可版本化
├─ IO List 与 Wiring Diagram 必须提交
├─ Test Data 与 Test Report 必须对应版本
├─ Controller Configuration 与参数必须备份
├─ Known Issues 必须随 Release Package 提交
└─ 每个付款节点应能对应一个可追踪版本记录

10. 最终确认原则

交付必须可运行、可复现、可接管、可追踪。
导航交付必须对齐附件一-A。
嵌入式交付必须对齐附件一-B。
阶段交付必须对齐附件二 System Milestones。
知识产权、背景技术和第三方组件必须按本附件披露。
未披露的限制，不得作为限制甲方使用、二次开发或商业化的理由。
