1. 文件定位

本文件定义 SnowBot Alpha/Beta 阶段最高层整机系统里程碑，用于统一导航、嵌入式、电控、机械、电气、测试和供应商交付节奏。

本文件不替代具体技术需求。专项技术开发应以对应文件为准：

├─ 导航与自主系统：Navigation & Autonomy Stack v2.1
├─ 嵌入式与电控系统：Embedded & Electromechanical Control Stack v2.1
├─ 机械结构：机械结构需求或专项 SOW
├─ 电气/BMS/配电：电气或电源专项 SOW
└─ 测试与演示：测试计划或演示计划

---

2. 里程碑层级

Milestone Hierarchy
├─ A/B Gate：整机系统里程碑
│  └─ 本文件定义，用于全项目推进
│
├─ P Gate：合同付款里程碑
│  └─ 主合同定义，用于付款和商务验收
│
└─ M Gate：专项技术里程碑
   ├─ 导航文档定义 M1-M5
   └─ 嵌入式文档按 A/B Gate 映射执行

解释原则：

├─ 整机阶段以本文件 A1-A4 / B1-B5 为准
├─ 合同付款以主合同 P0-P6 为准
├─ 导航交付以导航 v2.1 为准
├─ 嵌入式交付以嵌入式 v2.1 为准
└─ 供应商责任不得因本文件整机表述被扩大到其专项范围之外

3. Alpha · Prototype / Assist

Alpha 目标：

工程原型机可安全运行，基础作业可人工控制，导航与嵌入式链路可联调。

A1 系统架构冻结

类别内容
结构总体布局、底盘方案、除雪模块接口
嵌入式控制器、IO、通信架构
电气电池、BMS、配电架构
软件ROS2、控制框架
Gate可进入采购、加工、装配

输出：

├─ 总体布局冻结
├─ 底盘方案冻结
├─ 除雪模块接口冻结
├─ 控制器与通信架构冻结
├─ IO点表初版
├─ 电池/BMS/配电方案冻结
├─ ROS2与上位机控制框架冻结
└─ 关键采购清单冻结

A2 底盘与驱动系统打通

类别内容
结构底盘、轮组/履带、驱动安装
嵌入式左右驱动控制、编码器反馈
电气驱动器、电池、配电
Gate可前进、后退、转向、停车

输出：

├─ 底盘机械安装完成
├─ 驱动器与电源链路可运行
├─ 上位机或控制器可控制底盘运动
├─ 编码器/底盘反馈可读取
├─ 遥控器可基础控制
├─ 急停或停车链路有效
└─ 基础运行测试记录

A3 除雪作业模块完成

类别内容
结构前端安装结构、作业支撑结构
嵌入式作业机构控制
电气作业电机、供电
Gate可完成基础推雪 / 除雪动作

输出：

├─ 除雪作业机构安装完成
├─ 作业电机供电链路完成
├─ 成品控制器或作业控制链路完成
├─ 遥控器或上位机可启停作业机构
├─ 作业机构状态可读取或明确不支持项
├─ 基础除雪动作测试通过
└─ 作业机构联调测试记录

A4 Alpha 工程原型机完成

类别内容
整机结构、电气、嵌入式、基础软件
测试基础运行测试、基础除雪测试
Demo工程演示视频
Gate人可以安全控制机器完成基础作业

输出：

├─ 整机基础装配完成
├─ 底盘、电气、嵌入式、基础上位机联调完成
├─ 人工控制可完成基础移动
├─ 人工控制可完成基础除雪动作
├─ 急停/遥控接管/软件停车有效
├─ 基础日志或测试记录可复盘
├─ Alpha已知问题清单
└─ 工程演示视频

4. Beta · Task / Product Demo

Beta 目标：

固定场地内形成基础任务闭环，机器可按定义路线执行，遇到异常优先安全停车，并具备系统级演示能力。

B1 产品化结构重构

类别内容
结构模块化、一体化、装甲、维护接口
ID工业设计、整体产品感
Gate整机具备产品感，而非拼装原型状态

输出：

├─ 模块化结构方案
├─ 外壳/装甲/维护接口方案
├─ 传感器和控制器安装优化
├─ 线束维护路径建议
├─ 工业设计方向
└─ 产品展示状态改进清单

B2 定位与路线系统完成

类别内容
导航RTK、IMU、Odometry
软件Teach & Repeat
嵌入式运动执行接口
Gate可录制、保存、回放路线

输出：

├─ RTK/IMU/Odometry 数据可用
├─ 位置和姿态可解析
├─ 固定场地路线可录制
├─ 路线可保存、加载、回放
├─ 嵌入式可稳定执行路线回放控制指令
├─ 运动执行状态可反馈
└─ B2测试报告

B3 任务执行闭环完成

类别内容
导航路径跟踪
软件任务状态机
嵌入式速度执行、转向执行
安全障碍停车、降级逻辑
Gate固定场地内可按路线完成基础作业

输出：

├─ 固定路线低速路径跟踪
├─ 任务启动/暂停/停止/完成状态清晰
├─ 速度执行与转向执行稳定
├─ 基础障碍检测触发安全停车
├─ 定位/传感器/CAN/上位机异常时降级或停车
├─ 除雪机构状态可纳入任务状态机
├─ 人工接管优先
├─ 急停最高优先级
└─ B3联合测试报告

说明：Beta 阶段最低障碍策略为安全停车；局部避障为可选增强能力，除非合同另行明确。

B4 Beta 展示状态完成

类别内容
结构外壳、装甲、维护结构
测试夜间测试、低温测试、雪地测试
Demo展示流程、展示视频
Gate整机达到系统演示状态

输出：

├─ 外观与产品感达到展示要求
├─ 展示流程稳定
├─ 夜间/低温/雪地测试记录
├─ 展示视频素材
├─ 展示前问题清单
└─ 演示风险清单

说明：B4 是整机展示状态 Gate，不应被解释为单一导航或嵌入式供应商独立责任。

B5 System Demonstration & Engineering Handover

类别内容
功能固定场地路线执行、Teach & Repeat、基础障碍停车、人工接管 / 急停、日志记录、产品展示状态
Gate可向投资人、合作伙伴、客户进行系统级演示，并完成工程交付

输出：

├─ 固定场地路线执行演示
├─ Teach & Repeat 演示
├─ 基础障碍停车演示
├─ 人工接管/急停演示
├─ 除雪机构启停演示
├─ 运行日志记录与导出
├─ 导航工程交付
├─ 嵌入式工程交付
├─ 测试报告与已知问题清单
└─ 系统级演示准备完成

8. 最终原则

一个整机里程碑，多个专项交付。
A/B Gate 管全局节奏，P Gate 管付款，专项需求管技术交付。
不能让供应商自建里程碑打乱整机推进。
不能用整机语言扩大单一供应商责任。
不能用模糊“控制”“电控”“自动驾驶”替代可验收交付物。
