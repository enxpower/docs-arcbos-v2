2. SnowBot整机全貌

SnowBot是一台48V LFP低温户外履带除雪机器人。它不是简单履带底盘，也不是普通AGV，而是带除雪总成、高电流负载、堵转风险、低温环境和无人值守安全状态机的商业设备。

整机系统分工

A包：电池能源包
└─ 安全、稳定、可诊断地输出48V主电源。

B包：整车电控总包
└─ 让机器动起来、除起来、停下来、报故障。

C包：产品化电气工程包
└─ 把Alpha跑通的电气系统产品化，包含Dock、正式线束、低压、EMC。

I包：ARCBOS总架构包
└─ 定义系统、控制接口、验收和最终冻结。

B包必须理解整机安全边界。VCU不是普通控制板，而是整机状态边界和负载仲裁的执行者。

4. 除雪总成主电机要求

除雪总成主电机不是单纯绞龙电机。它承担进雪、破雪、输送、加速雪流、抛雪、堵转恢复和急停能量处理。

项目目标说明
连续功率2.5–3.5kW主工作区
重载功率4–5kW湿雪、厚雪、压实雪
峰值功率5–6kW突然进雪、半堵转
控制CAN必须可由VCU控制并反馈状态
功能软启动、限流、堵转检测、反转清堵、热降额必须提供策略说明
低温-30°C验证冷启动必须测试
防护IP65或更高含电机出线和连接器

不允许直接把5kW定义为连续功率。5kW只能作为重载或峰值设计范围的一部分，最终必须由负载谱和传动方案验证。

5. 除雪总成传动输入

B包必须与机械结构方协同确认传动方案。未确认传动方案前，不得冻结电机。

传动方案必须说明

转速与传动
├─ 叶轮目标转速
├─ 绞龙目标转速
├─ 传动比
└─ 是否同轴

清堵与保护
├─ 是否允许反转
├─ 反转时叶轮是否随动
├─ 是否有机械过载保护
└─ 堵转时控制器 / BMS / 机械保护动作顺序

安全停机
├─ 急停后叶轮停止时间
└─ 惯量释放方式

推荐优先评估一台主电机通过传动比分配到叶轮和绞龙的方案。不建议直接冻结绞龙和叶轮同轴同速刚性直连。

6. 堵转与反转清堵要求

堵转恢复流程

检测电流上升和转速下降
    ↓ 100–200ms
控制器限流
    ↓
停止正转
    ↓
反转清堵300–500ms
    ↓
再次正转软启动
    ↓
若恢复，继续低速作业
    ↓
若再次堵转，最多再尝试1次
    ↓
两次失败后进入FAULT_STOP

堵转不能靠熔断器处理。普通堵转必须由控制器限流、VCU判断、反转清堵和状态机共同处理。

场景预期母线电流动作策略
湿雪软堵转150–250A限流 + 反转清堵
冰雪混合半堵转250–350A快速限流 + 停转判断
硬冰或异物完全堵转350–500A瞬态保护，不允许硬顶
控制器限流后堵转200–280A热降额或停机

硬规则：

VCU硬规则

CHARGING
└─ 禁止履带和除雪总成动作。

SERVICE
└─ 禁止自动动作。

E_STOP
└─ 优先级最高。

通信丢失
└─ 不得维持危险动作。

低压异常
└─ 必须进入降级或停机。

9. NAV/COMMS降级动作

导航与通信状态

NAV_OK
└─ 定位正常，允许正常作业。

NAV_DEGRADED
└─ 降速，保持安全距离，禁止靠近车辆、边缘、坡道高速动作。

NAV_LOST
└─ 停止除雪总成，低速停车或回最后安全点，禁止继续新区域作业。

COMMS_LOST
└─ 本地定位正常时可完成当前安全段，不进入新区域或高风险任务。

LOCAL_SAFE
└─ 安全停车，记录日志，等待恢复。

该逻辑必须进入VCU，不得仅交给导航软件或云端处理。

10. 满电下坡策略

满电下坡是硬规则，不是运营建议。

条件VCU动作
SOC大于90%且BMS.ChargeAllowed为FALSE禁止进入坡度大于5%且长度大于20m的下坡路段
SOC大于95%且BMS.RegenAllowed为FALSE履带回生功率限制为不超过500W或供应商允许的更低值
已在坡道且母线电压接近过压阈值进入DERATE，降低速度，机械制动主导
母线过压触发受控停车，记录故障，禁止继续下坡

11. 负载仲裁要求

负载仲裁规则

除雪总成重载
└─ 履带速度自动降低。

履带原地转向
└─ 除雪总成不得处于满功率进雪状态。

PTC预热
└─ 不得与重载除雪同时满功率运行。

主母线电流持续超过300A
└─ 进入DERATE。

峰值超过BMS允许值
└─ 进入FAULT_STOP或受控停机。

B包必须提供负载仲裁逻辑说明，不能让每个控制器各自拉满。

19. B包完整责任覆盖与技术要求

本节明确B包必须承接的完整技术责任。B包不是单纯电机采购包，也不是普通嵌入式外包包，而是Alpha阶段整车运动、除雪、状态机和故障闭环的核心总包。

主方案主题B包责任要求

19.2 整机负载谱控制策略

B包必须把主方案L0–L7负载谱转化为控制策略。

负载谱到控制策略

L0/L1：待机与上电
└─ VCU自检、通信检查、禁止高风险动作。

L2：低温预热
└─ PTC与动力负载不得同时拉满，需与A包BMS温度状态联动。

L3：空载行走
└─ 履带正常控制，记录行走电流基线。

L4：正常除雪
└─ 允许SNOW_WORK，记录除雪总成和履带叠加电流。

L5：重载除雪
└─ 进入重载控制，降低履带速度或除雪进给速度。

L6：短时峰值
└─ 允许短时峰值，但必须受BMS允许值、控制器限流和时间窗口约束。

L7：保护区
└─ 不作为工作区，进入DERATE、FAULT_STOP或受控停机。

19.3 辅助执行器要求

B包必须覆盖辅助执行器，不得把旋转、俯仰、抬升视为无关小电机。

辅助执行器要求

旋转电机
├─ 100–200W级方向
├─ 带减速
├─ 带位置反馈或限位
├─ 有堵转保护
└─ 进入VCU状态机

俯仰电机
├─ 100–200W级方向
├─ 必须有自锁、抱持或制动能力
├─ 断电后不得自由下落
├─ 有行程限位
└─ 进入VCU状态机

抬升执行器
├─ 8000N、8mm/s、120mm行程可作为候选目标
├─ 必须验证低温效率
├─ 必须验证自锁
├─ 必须验证过载保护
├─ 必须验证冰雪卡滞
└─ 必须验证寿命循环

19.4 硬冰层边界必须进入VCU

B包不得把硬冰层问题留给运营口头说明。VCU必须对持续高阻力、堵转和疑似硬冰做状态处理。

硬冰处理规则

普通积雪 / 新雪
└─ 正常SNOW_WORK。

湿雪 / 厚雪 / 轻度压实雪
└─ 降速、限流、必要时多次处理。

1cm以下薄冰
└─ 可短时尝试扰动，但必须受峰值保护约束。

1–3cm硬冰
└─ 高风险，短时尝试失败后降级或绕行。

3cm以上硬冰
└─ 不作为工作对象，不硬顶，进入FAULT_STOP、LOCAL_SAFE或绕行策略。

19.5 急停与安全停机要求

B包必须配合硬件急停链路，并定义受控停机过程。

急停动作链

急停触发
    ↓
硬件切断驱动与除雪使能
    ↓
VCU记录状态
    ↓
电机控制器进入安全扭矩关闭或等效状态
    ↓
机械制动进入保持
    ↓
必要时请求A包BMS断开主接触器

B包不得把急停只做成软件按钮。B包必须说明急停后履带、除雪总成、辅助执行器和制动器分别如何动作。

19.6 初版低压与线束边界

Alpha阶段B包负责初版低压与线束，是为了让样机跑起来；Beta产品化阶段必须把正式线束、低压配电、EMC工程移交C包。

B包Alpha初版低压/线束责任

必须做到
├─ 48V到24V/12V基本供电
├─ VCU、控制器、传感器基本上电
├─ CAN通信可用
├─ 急停链路可用
├─ 线束安全固定
├─ 高低压基本分区
└─ 支持Alpha测试

不得声称完成
├─ 正式线束图冻结
├─ 产品化端子表冻结
├─ EMC正式通过
├─ 产品化接地屏蔽冻结
└─ 批量装配工艺冻结

20. 辅助执行器、负载仲裁与传动低温强制要求

本节定义B包必须满足的辅助执行器测试、负载仲裁量化、传动件低温材料、Alpha到Beta移交和新增否决要求。供应商必须按本节提交可执行、可测试、可验收的工程资料。

20.1 辅助执行器测试与验收要求

辅助执行器不是附属小项，必须进入B包测试交付物和验收标准。

辅助执行器测试清单

旋转电机
├─ 正反转命令响应测试
├─ 左右极限位置测试
├─ 限位触发后停止测试
├─ 堵转保护测试
├─ 低温启动测试
└─ VCU故障码记录测试

俯仰电机
├─ 上仰/下俯命令响应测试
├─ 行程限位测试
├─ 断电自锁或抱持测试
├─ 断电后不得自由下落测试
├─ 堵转保护测试
├─ 低温卡滞测试
└─ VCU故障码记录测试

抬升执行器
├─ 8000N目标载荷能力验证或供应商等效证明
├─ 8mm/s速度验证或供应商等效证明
├─ 120mm行程验证或供应商等效证明
├─ 低温效率测试
├─ 自锁测试
├─ 过载保护测试
├─ 冰雪卡滞测试
└─ 寿命循环测试计划

验收标准：辅助执行器必须能被VCU控制、能反馈状态或限位、能在异常时停止、能记录故障码。未进入VCU状态机的辅助执行器不得进入Alpha装机。

20.2 负载仲裁量化阈值初版

以下阈值为Alpha阶段初始建议值，用于让VCU软件团队有可执行边界。最终阈值必须由负载谱测试、BMS能力、控制器曲线和I包评审冻结。

触发条件VCU动作退出条件
主母线电流持续超过270A超过3秒进入DERATE；限制除雪总成功率至2.5kW级；履带速度降至0.3m/s级主母线电流低于220A持续5秒
主母线电流超过320A持续超过1秒进入FAULT_STOP或受控停机人工复位或I包批准的恢复流程
除雪总成进入堵转清堵状态禁止履带原地转向；维持低速前进或停止堵转清堵完成且电流恢复正常
PTC预热功率大于500W限制除雪总成启动或禁止进入SNOW_WORKPTC降至维温功率或BMS温度达到允许区间
BMS DischargeAllowed为FALSE禁止履带和除雪总成新动作，进入安全停机BMS恢复允许且故障被确认清除
BMS RegenAllowed为FALSE且SOC大于95%限制或禁止履带回生，机械制动主导SOC下降或BMS恢复RegenAllowed

B包可以提出不同阈值，但必须提交工程依据。未经I包批准，软件工程师不得自行决定关键阈值。

20.3 传动件低温材料与冷启动要求

B包必须把传动件低温材料纳入设计交付和测试交付。

传动件低温材料要求

皮带方案
├─ 必须提供-40°C或更低温度等级材料资料
├─ 必须提供低温张力变化说明
├─ 必须验证-30°C冷启动不打滑或不过度打滑
└─ 不接受只按常温工业皮带选型

链条方案
├─ 必须提供低温润滑脂资料
├─ 润滑脂倾点建议≤-50°C或满足-40°C工作说明
├─ 必须验证低温张紧器动作
└─ 必须考虑盐水、雪泥、颗粒磨损

齿轮箱方案
├─ 必须提供低温合成齿轮油资料
├─ 必须说明-30°C冷启动粘度影响
├─ 必须检查密封件低温收缩泄漏
└─ 必须验证低温空载和加载启动电流

20.4 传动件冷启动测试

传动件冷启动测试

测试准备
├─ -30°C浸泡48小时
├─ 保持实际润滑脂/齿轮油/皮带材料
├─ 保持实际张紧结构
└─ 保持实际电机控制器限流设置

启动流程
├─ 首次低速空转30–60秒
├─ 记录启动电流
├─ 记录RPM响应
├─ 记录异常噪声和振动
├─ 逐步加载模拟雪负载
└─ 记录热降额和故障码

判定
├─ 不允许卡死
├─ 不允许电流异常持续高位
├─ 不允许皮带明显打滑后无法恢复
├─ 不允许链条跳齿或张紧失效
└─ 不允许齿轮箱泄漏或异常噪声持续

20.5 Alpha到Beta低压与线束移交条件

B包初版低压/线束不能被C包直接当作产品化冻结输入。必须满足以下条件后，才允许C包基于B包输入开展正式线束设计。

B包向C包移交触发条件

文件条件
├─ IO点表已经过I包评审确认
├─ VCU状态机初版已通过R2评审
├─ 除雪总成控制器端子定义已确认
├─ 左右履带控制器端子定义已确认
├─ VCU端子定义已确认
├─ 辅助执行器端子定义已确认
└─ 初版CAN拓扑已确认

测试条件
├─ Alpha样机完成基本上电测试
├─ Alpha样机完成状态切换测试
├─ 履带基本动作测试完成
├─ 除雪总成空载动作测试完成
└─ 急停链路测试完成

批准条件
└─ I包正式签发“C包可以启动产品化电气工程”的书面通知。

20.6 B包新增一票否决项

B包新增一票否决项

1. 辅助执行器未进入VCU状态机且无测试记录。
2. 辅助执行器无端子定义、控制接口或故障反馈说明。
3. 传动件未提供低温材料或低温润滑资料。
4. 不做-30°C传动件冷启动测试。
5. 负载仲裁阈值未量化，由软件工程师自行决定。
6. B包初版线束未经I包批准即移交C包产品化。
