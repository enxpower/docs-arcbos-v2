《保密、知识产权归属、竞争限制及禁止绕过协议》

英文名称：Shanghai Arcbos Systems Ltd

品牌：ARCBOS

英文释义：Autonomous Robotic Core for Brutal Outdoor Systems

注册地址：上海市青浦区公园路348号7层

乙方：

身份证号/统一社会信用代码：

联系方式：

鉴于：

  1. 甲方系从事工业级机器人、自动驾驶、人工智能、嵌入式系统、机器人操作系统、机器人云平台及相关技术研发、商业化与产业化的科技公司；
  2. 乙方可能以员工、顾问、Freelancer、外包方、项目开发者、联合研发方、供应商或其他合作身份参与甲方业务；
  3. 在合作过程中，乙方将接触大量涉及甲方核心竞争力、商业秘密、技术资产及知识产权的信息与成果；
  4. 为明确双方权利义务，保护甲方技术资产、商业利益、客户资源、供应链资源、投资资源及知识产权，根据《中华人民共和国民法典》《中华人民共和国劳动合同法》《中华人民共和国反不正当竞争法》《中华人民共和国著作权法》《中华人民共和国专利法》等相关法律法规，双方达成本协议。

第一条 定义条款

1.1 保密信息

“保密信息”系指甲方以任何形式向乙方披露、提供、展示、发送、共享、开放、授权访问或允许接触的全部非公开信息，包括但不限于：

（一）技术信息；

（二）商业信息；

（三）研发信息；

（四）软件与代码；

（五）算法与模型；

（六）客户与供应链信息；

（七）投资人与融资信息；

（八）测试与运营数据；

（九）未公开商业计划；

（十）甲方认为具有商业价值或保密必要性的其他信息。

保密信息无论是否标注“保密”“Confidential”“内部”“Restricted”等字样，均受本协议保护。

1.2 技术资料

“技术资料”包括但不限于：

（一）源代码、脚本、SDK、API、二进制程序；

（二）GitHub、GitLab、Gitea、Bitbucket 等仓库内容；

（三）机器人系统架构；

（四）SLAM、导航、路径规划、控制算法；

（五）AI模型、训练数据、推理逻辑；

（六）CAD、STEP、STP、SolidWorks、Creo、NX、Fusion 文件；

（七）PCB、原理图、BOM、电气图纸；

（八）嵌入式固件、MCU程序；

（九）测试数据、日志、录像、地图；

（十）机械结构、装配方案、接口定义；

（十一）机器人操作系统；

（十二）Mission Planning、Fleet Management 架构；

（十三）Notion 文档；

（十四）产品路线图；

（十五）技术方案、草稿、白板记录；

（十六）尚未完成的研发成果。

1.3 商业秘密

“商业秘密”包括符合以下条件的信息：

（一）不为公众所知悉；

（二）具有商业价值；

（三）甲方已采取合理保密措施。

包括但不限于：

  • 技术秘密；
  • 工艺流程；
  • 软件架构；
  • 算法逻辑；
  • 机器人结构设计；
  • BOM；
  • 成本结构；
  • 供应链体系；
  • 客户资料；
  • 商务报价；
  • 投资人与融资信息；
  • 商业计划；
  • 市场策略；
  • AI训练数据；
  • 系统集成方案。

第二条 保密义务

2.1

乙方对甲方全部保密信息承担严格保密义务。

未经甲方书面同意，乙方不得：

（一）复制；

（二）传播；

（三）披露；

（四）出售；

（五）转授权；

（六）共享；

（七）展示；

（八）下载至未经授权环境；

（九）用于本协议目的以外事项。

2.2

乙方不得以任何形式向第三方透露：

  • 甲方技术路线；
  • 产品架构；
  • 研发进度；
  • 投资计划；
  • 客户情况；
  • 供应链信息；
  • 内部管理信息。

2.3

乙方应采取不低于行业合理标准的安全措施保护甲方信息，包括但不限于：

  • 权限隔离；
  • 本地加密；
  • 访问控制；
  • 多因素认证；
  • 不使用公共网盘；
  • 不向私人邮箱转发；
  • 不向未经授权设备同步。

第三条 知识产权归属

3.1

乙方在合作期间基于：

  • 甲方项目；
  • 甲方资料；
  • 甲方需求；
  • 甲方技术路线；
  • 甲方资金；
  • 甲方业务方向；

所形成的全部成果，其知识产权、使用权、收益权、申请权、处分权均归甲方独占所有。

3.2

前述成果包括但不限于：

  • 已完成成果；
  • 未完成成果；
  • 半成品；
  • 草稿；
  • Demo；
  • 中间版本；
  • 推演方案；
  • 白板记录；
  • 临时代码；
  • AI生成内容；
  • 测试版本；
  • 设计思路。

3.3

无论乙方是否：

  • 在家开发；
  • 使用个人电脑；
  • 使用个人账号；
  • 使用个人设备；
  • 非工作时间开发；

只要与甲方项目、技术方向、业务路线存在关联，均视为职务成果或合作成果，知识产权归甲方所有。

第四条 AI生成成果归属

4.1

乙方利用：

  • ChatGPT；
  • Claude；
  • Gemini；
  • Copilot；
  • Cursor；
  • DeepSeek；
  • 任意生成式AI工具；

形成的与甲方业务相关成果，其全部知识产权归甲方所有。

4.2

乙方不得主张：

  • AI生成代码属于公共领域；
  • AI生成成果无归属；
  • AI工具平台拥有优先权；
  • 乙方拥有独立版权。

4.3

乙方应确保：

  • 不向第三方AI平台泄露甲方商业秘密；
  • 不上传敏感源码；
  • 不上传未公开架构；
  • 不上传客户信息；
  • 不上传供应链信息。

第五条 GitHub与代码资产归属

5.1

甲方GitHub组织、仓库、Issue、Pull Request、Commit、Wiki、Release、Action、CI/CD、Branch、Tag、Discussion、Package、Secret、Token 等全部属于甲方资产。

5.2

乙方提交的：

  • Commit；
  • Branch；
  • Fork；
  • Pull Request；
  • Script；
  • Automation；

均归甲方所有。

5.3

GitHub提交记录、Issue记录、Pull Request记录、Review记录、CI/CD记录可作为：

  • 开发事实证据；
  • 权属证据；
  • 商业秘密证据；
  • 违约证据；
  • 司法证据。

5.4

乙方不得：

（一）私自Fork核心仓库；

（二）复制代码库；

（三）导出完整仓库；

（四）保留离职后访问权限；

（五）绕过权限控制；

（六）删除审计记录；

（七）篡改提交历史。

第六条 CAD、STEP及图纸归属

6.1

全部：

  • CAD文件；
  • STEP文件；
  • STP文件；
  • 图纸；
  • PCB；
  • BOM；
  • 工艺文件；
  • 装配结构；

均属于甲方独占资产。

6.2

乙方不得：

  • 私自保存；
  • 用于其他项目；
  • 向第三方展示；
  • 申请专利；
  • 用于竞争产品；
  • 用于个人创业。

第七条 数据与测试结果归属

7.1

全部：

  • 测试数据；
  • 地图数据；
  • 运行日志；
  • AI训练数据；
  • 传感器数据；
  • 视频录像；
  • 标注数据；
  • 故障记录；

均归甲方所有。

7.2

乙方不得：

  • 私自复制；
  • 留存；
  • 训练其他模型；
  • 提供第三方；
  • 用于竞争产品。

第八条 专利与软著限制

8.1

未经甲方书面同意，乙方不得：

（一）申请专利；

（二）申请软件著作权；

（三）申请商标；

（四）申请论文发表；

（五）申请技术成果登记。

8.2

乙方不得以：

  • 个人名义；
  • 第三方名义；
  • 关联公司名义；

申请与甲方业务相关知识产权。

第九条 禁止私自竞争与商业化

9.1

乙方不得利用：

  • 甲方技术；
  • 甲方结构；
  • 甲方算法；
  • 甲方系统架构；
  • 甲方供应链；
  • 甲方客户资源；

从事与甲方存在竞争关系的业务。

9.2

包括但不限于：

  • 自行创业；
  • 参与竞争公司；
  • 提供顾问服务；
  • 技术转让；
  • 联合开发；
  • ODM/OEM支持。

第十条 竞业限制

10.1

对于员工、高级顾问、核心研发人员、掌握商业秘密人员，自合作终止之日起二十四个月内，未经甲方书面同意，不得：

（一）任职于竞争企业；

（二）投资竞争企业；

（三）成立竞争企业；

（四）为竞争企业提供技术支持；

（五）从事与甲方核心业务实质相同业务。

10.2

竞争业务包括但不限于：

  • 工业机器人；
  • 除雪机器人；
  • 室外无人机器人；
  • SLAM；
  • 机器人导航；
  • 边缘AI；
  • 多机器人协同；
  • 工业自动驾驶；
  • 机器人云平台；
  • 工业自动化。

10.3

甲方可依据法律规定向适用竞业限制的乙方支付竞业限制补偿。

补偿标准可由：

  • 单独协议；
  • Offer；
  • 服务合同；
  • 补充协议；

另行约定。

10.4

如乙方违反竞业限制义务，应：

（一）返还全部竞业补偿；

（二）承担违约金；

（三）赔偿全部损失；

（四）停止侵权行为。

第十一条 禁止绕过

11.1

乙方不得绕过甲方，直接或间接联系：

  • 客户；
  • 供应商；
  • 制造商；
  • 投资人；
  • 渠道方；
  • 合作伙伴。

11.2

乙方不得利用甲方资源建立自身商业关系。

第十二条 禁止挖人

12.1

乙方在合作期间及终止后二十四个月内，不得：

  • 挖角；
  • 招募；
  • 诱导离职；
  • 共同创业；
  • 私下合作；

涉及甲方员工、顾问、供应商、合作伙伴。

第十三条 禁止接触客户与供应链

13.1

未经授权，乙方不得：

  • 接触甲方客户；
  • 使用客户资料；
  • 使用供应链信息；
  • 获取报价体系；
  • 建立私下交易。

第十四条 投资人与融资保护

14.1

乙方不得：

  • 绕过甲方联系投资人；
  • 利用甲方BP融资；
  • 使用甲方融资材料；
  • 使用甲方估值逻辑；
  • 对外宣称与甲方存在未授权关系。

第十五条 资料返还与删除

15.1

合作终止后，乙方应立即返还：

  • 文件；
  • 图纸；
  • 代码；
  • 存储介质；
  • 账号；
  • 密钥；
  • Token；
  • 测试数据；
  • 样机资料。

15.2

乙方应删除：

  • 本地文件；
  • 云盘文件；
  • 邮箱附件；
  • 聊天缓存；
  • 私有备份；
  • Git本地副本。

15.3

甲方有权要求乙方提供：

  • 删除证明；
  • 系统截图；
  • 销毁确认；
  • 书面声明。

第十六条 GitHub权限回收

16.1

甲方有权随时：

  • 关闭权限；
  • 删除账号；
  • 撤销访问；
  • 收回Token；
  • 收回SSH Key。

16.2

乙方不得在权限终止后继续访问甲方系统。

第十七条 电子证据条款

17.1

双方确认以下内容可作为合法证据：

  • GitHub记录；
  • Git提交记录；
  • Pull Request；
  • CI/CD日志；
  • Notion记录；
  • 飞书记录；
  • 微信聊天；
  • 邮件；
  • Slack；
  • Teams；
  • 服务器日志；
  • 文件元数据；
  • 云端访问日志；
  • 录音录像。

17.2

电子数据与系统日志具有与书面文件同等证据效力。

第十八条 违约责任

18.1

乙方违反本协议的，应承担：

（一）违约金；

（二）实际损失；

（三）预期利益损失；

（四）律师费；

（五）调查费；

（六）公证费；

（七）鉴定费；

（八）保全费。

18.2

如损失难以计算，甲方有权按以下较高者主张赔偿：

（一）乙方违法所得；

（二）甲方实际损失；

（三）人民币100万元起的违约金；

（四）相关项目合同金额三倍。

第十九条 紧急禁令权

19.1

甲方有权针对：

  • 泄密；
  • 侵权；
  • 竞争；
  • 数据转移；
  • 客户绕过；

申请行为保全、证据保全、禁令及其他司法措施。

第二十条 持续有效性

20.1

保密义务不因：

  • 离职；
  • 项目结束；
  • 合同终止；
  • 合作终止；

而终止。

20.2

商业秘密保护期限持续至相关信息合法公开之日止。

第二十一条 法律适用

21.1

本协议适用中华人民共和国法律。

第二十二条 争议解决

22.1

因本协议引起的争议，双方应优先协商解决。

22.2

协商不成的，任一方有权向甲方所在地有管辖权人民法院提起诉讼。

第二十三条 可分割性条款

23.1

如本协议部分条款被认定无效，不影响其他条款效力。

第二十四条 协议效力

24.1

本协议自双方签署之日起生效。

24.2

电子签署、电子邮件确认、在线签署系统确认均具有同等法律效力。

第二十六条 最终条款

26.1

（一）已充分阅读本协议；

（三）知悉甲方核心业务高度依赖商业秘密与技术资产；

（四）认可本协议对甲方商业利益保护的必要性与合理性。
