ENGINEERED FOR EXTREME CONDITIONS

文件定位

本文件用于发给第三方软件 / 导航 / 机器人开发商。

目的不是直接报价，而是请开发商基于 ARCBOS 已定义的 Alpha / Beta 阶段产品远景，输出其认为合理的软件开发清单、模块拆分、依赖条件、风险点和里程碑建议。

本文件不是正式 SOW，不是合同附件，也不是最终报价范围。

正式报价应在双方完成开发清单对齐后，基于确认后的 SOW 另行提交。

当前阶段原则

SnowBot 当前只推进两个阶段：

本轮征询只围绕 Alpha / Beta 阶段展开。

请勿将后续 Gamma / Delta / Fleet / Service 阶段能力默认纳入当前清单或报价。

产品背景

SnowBot 是 ARCBOS 面向极寒环境和商用除雪场景开发的工业级机器人。

当前阶段目标不是一次性完成全自主无人值守系统，而是先完成工程样机的软件基础能力与固定场地基础任务闭环。

目标场景包括：

  • 商业园区道路
  • 工业园区内部道路
  • 停车场关键通道
  • 校园 / 医院 / 物业场地
  • 固定路线或固定区域除雪作业

两阶段产品远景总览

Alpha阶段产品远景

阶段定义

Alpha 阶段的目标是：

产品状态

Alpha 阶段应达到：

  • 机器可以被人工安全控制
  • 底盘控制接口可用
  • 传感器数据可以接入
  • 工控机 / 软件环境可以运行
  • 急停和基础安全链路可用
  • 具备进入 Beta 阶段自动任务闭环开发的基础

Alpha阶段能力树

Alpha阶段请开发商重点评估

请开发商评估以上能力是否完整，并补充遗漏项，重点包括：

  • 是否缺少必要底盘接口
  • 是否缺少必要传感器接口
  • 是否缺少关键安全联动
  • 是否缺少必要部署或调试工具
  • 是否存在影响后续 Beta 闭环的前置依赖

Alpha阶段不追求

  • 自动作业
  • 自动路径规划
  • SLAM / 建图
  • 全局路径规划
  • 自动回充
  • 云平台
  • 多机协同
  • 无人值守

Beta阶段产品远景

阶段定义

Beta 阶段的目标是：

产品状态

Beta 阶段应达到：

  • 人可以定义或录制一条路线
  • 机器人可以在固定场地内重复执行该路线
  • 机器人可以进行基础定位与路径跟踪
  • 遇到基础障碍物时优先安全停车
  • 出现定位或传感器异常时可降级或停车
  • 能输出日志、问题记录和交付文档
  • 可用于工程演示、客户初步交流和后续融资展示

Beta阶段能力树

Beta阶段请开发商重点评估

请开发商评估以上能力是否完整，并补充遗漏项，重点包括：

  • 定位融合方案是否足够支撑固定路线回放
  • 路线录制 / 回放机制是否可交接、可复现
  • 路径跟踪控制是否需要底盘侧配合
  • 基础障碍检测以“停车”为主是否合理
  • 降级逻辑是否完整
  • 任务状态机是否覆盖必要状态
  • 是否缺少现场联调或数据复盘工具

Beta阶段不追求

  • 复杂 SLAM / 建图硬交付
  • 全局路径规划
  • 区域覆盖路径规划
  • 复杂动态避障
  • 自动回充 / 自动对桩
  • 云端任务调度
  • Fleet 多机协同
  • 无人值守
  • 产品级完整 HMI
  • 长期运营维护系统
