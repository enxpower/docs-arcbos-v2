研发规则与知识产权

PUB-2604-0003-DOC · 1.0

各位：

即日起，项目研发统一进入公司仓库管理，执行以下规则：

内部团队规则

一、唯一交付原则

所有设计文件、代码、图纸，必须提交至公司指定仓库。

未进入仓库的内容，视为未完成、未交付。

二、提交频率

每周至少一次正式提交（建议周五前完成）。

禁止一次性集中交付，必须持续提交。

三、提交内容要求

每次提交必须包含：

1）实际成果文件（代码 / 图纸 / STEP / 文档）

2）清晰提交说明（说明改了什么）

四、文件规范

禁止使用以下命名：最终版 / 最新版 / 改改改

统一采用：模块_内容_版本_日期

五、外包交付

外包开发必须按周提交至公司仓库。

未提交部分不计入交付与验收。

六、权限与归属

所有仓库归公司统一管理。

不得使用个人仓库作为主交付。

所有成果默认归公司所有。

七、进度认定

以仓库提交记录为唯一进度依据。

未提交即视为未完成。

请所有人员严格执行，本周开始生效。

外包开发与交付规则

一、成果归属

所有开发成果（代码、设计、图纸、文档等）自形成之日起即归公司所有，外包方不保留任何权利。

二、仓库交付

外包方必须在公司指定仓库进行开发或按周同步提交。

未进入公司仓库的内容，不视为交付成果。

三、提交频率

至少每周提交一次完整可用版本（代码 / 文件 / 文档）。

禁止项目结束后一次性打包交付。

四、交付标准

每次提交必须满足：

1）可打开 / 可运行（代码）

2）结构清晰、无缺失文件

3）附带必要说明（说明本次修改内容）

五、验收与付款

所有验收、里程碑确认及付款，以公司仓库中的提交记录为唯一依据。

未提交部分不计入验收范围。

六、知识与资料限制

外包方不得：

  • 私自留存核心代码或设计
  • 向第三方提供或复用本项目成果
  • 用于其他项目或商业用途

七、权限控制

仓库权限由公司统一分配与回收。

项目结束或合作终止时，立即撤销外包方全部访问权限。

八、违约责任

如未按约定提交或存在隐瞒、保留、外泄等行为，公司有权：

  • 拒绝付款
  • 要求赔偿损失
  • 追究法律责任

知识产权归属条款

  1. 成果归属

    乙方在本项目中形成的全部成果，包括但不限于源代码、软件程序、算法、设计图纸、结构模型、技术文档、数据及相关资料（统称“项目成果”），其全部知识产权及相关权益自形成之日起即归甲方所有。

  2. 范围界定

    前述项目成果包括：

    （1）在本合同履行过程中完成的成果；

    （2）基于甲方资料、需求或资源形成的成果；

    （3）对既有成果的修改、优化及衍生内容。

  3. 权利转让

    乙方同意将上述项目成果的全部权利（包括但不限于著作权、专利申请权、商业秘密权益等）无偿、永久、不可撤销地转让给甲方，且无需另行签署文件。

  4. 协助义务

    如甲方为申请专利、著作权登记或其他权利，需要乙方配合签署文件或提供说明，乙方应无条件配合。

  5. 不得使用

    未经甲方书面同意，乙方不得：

    （1）自行使用项目成果；

    （2）向第三方提供或披露；

    （3）用于其他项目或商业用途。

  6. 无权主张

    乙方不得就项目成果主张署名权以外的任何权利（如适用），亦不得以任何形式限制甲方对成果的使用、修改、转让或商业化。

  7. 违约责任

    如乙方违反本条约定，甲方有权要求停止侵害、赔偿损失，并追究法律责任。
