SnowBot Alpha补齐与Beta闭环移动机器人导航控制系统软件外包主需求文档

版本：v1.4

用途：第三方外包询价、技术评审、合同附件、交付验收基线

适用项目：SnowBot 工程样机 Alpha补齐 + Beta闭环 阶段

1. 项目背景

本项目仅针对 SnowBot 工程样机阶段的软件能力建设。

本次外包范围严格限定为：

不包含：

  • Gamma（有监督自动）
  • Delta（无人值守）
  • Fleet（多机协同）
  • Service（云平台/运营）

供应商可以提交完整路线图建议，但当前报价、合同、交付、验收均只按 Alpha补齐 + Beta闭环执行。

2. 阶段定义（强约束）

禁止跨阶段开发。

3. 一句话需求定义

外包方负责完成 SnowBot 在 Alpha补齐与Beta闭环阶段的软件系统建设，使工程样机具备“可联调、可运行、可执行固定任务、可演示、可验收、可交接”的能力。

4. 外包范围拆分

4.1 Alpha补齐（系统打通）

目标：让系统可运行、可联调。

包括：

  • IPC / 工控机环境搭建
  • ROS2 或等效运行环境部署
  • 底盘控制接口接入
  • 底盘反馈接口接入
  • RTK / GNSS 驱动接入
  • IMU 驱动接入
  • 里程计 / odometry 接入
  • lidar 接入
  • 手动控制链路打通
  • 手动 / 自动模式切换
  • 急停软件联动
  • 基础日志系统
  • 模块间通信打通
  • 基础部署脚本

交付标准：系统可运行、可联调、可基础控制。

4.2 Beta闭环（任务能力）

目标：完成固定场地任务执行闭环。

包括：

  • 定位融合（RTK + IMU + odometry）
  • 固定场地局部坐标系建立
  • 路线录制
  • 路线存储
  • 路线回放
  • 路径跟踪控制
  • 基础路径偏差处理
  • 状态机（idle / ready / record / replay / pause / stop / degraded / complete）
  • 基础障碍检测
  • 前向安全区逻辑
  • 边界约束逻辑
  • 安全停车（优先）
  • 降级逻辑
  • 任务执行控制（启动 / 暂停 / 恢复 / 停止）
  • 日志导出
  • 参数调整工具
  • 现场联调

交付标准：完成端到端运行闭环。

6. 明确排除范围

本次外包不包含：

  • SLAM / 建图
  • 全局路径规划
  • 区域覆盖算法
  • 自动回充
  • 自动对桩
  • 多机协同
  • 云平台
  • OTA
  • 高级动态避障
  • 无 GPS 兜底定位
  • 无人值守
  • 长期运维
  • 客户反馈系统
  • 整机最终工程责任
  • 整机最终安全认证责任

7. GitHub 与代码交付要求

所有正式交付必须进入 ARCBOS 指定 GitHub 仓库。

未进入指定仓库的内容，不视为正式交付。

要求：

  • 每周至少一次正式 commit
  • 提交完整源码
  • 提交配置文件
  • 提交启动脚本
  • 提交测试脚本
  • 提交部署说明
  • 提交接口文档
  • 提交已知问题清单
  • 提交外部依赖清单

禁止：

  • 只交付 demo
  • 只交付二进制
  • 使用个人仓库作为主交付
  • 隐藏核心逻辑
  • 使用未披露 SDK / API / 云服务
  • 写死个人路径、账号、token、服务器地址

代码要求：

  • UTF-8 编码
  • 英文变量名
  • 英文函数名
  • 英文文件名
  • 代码注释使用英文
  • README / 文档可中英文，但英文优先

8. 外部依赖与供应链安全

未经 ARCBOS 书面确认，第三方不得引入外部 API、SDK、云服务、地图服务、日志服务、监控服务或在线模型。

第三方必须提交外部依赖清单，包括：

  • 依赖名称
  • 版本号
  • 来源
  • 开源协议
  • 是否可商用
  • 是否联网
  • 是否上传数据
  • 是否影响北美部署
  • 是否限制源码交付

9. 知识产权与交付权利

ARCBOS 至少应拥有：

  • 完整源代码访问权
  • 永久内部使用权
  • 后续修改权
  • 二次开发权
  • 集成权
  • 内部部署权
  • 文档与测试材料使用权

第三方不得：

  • 只交付黑盒系统
  • 将核心功能锁定在第三方私有环境
  • 要求后续维护必须依赖第三方人员
  • 未披露即引入受限授权组件

10. 里程碑

M1：接口与架构冻结

交付：

  • 软件架构
  • 模块边界
  • 接口清单
  • 依赖清单
  • 部署环境定义

验收重点：接口清晰、范围冻结、不越阶段。

M2：Alpha补齐完成（系统可运行）

交付：

  • 工控机 / ROS2 环境可运行
  • 底盘控制接口接入
  • 传感器输入接入
  • 急停软件联动
  • 基础日志可用

验收重点：系统能启动、能通信、能基础控制、能联调。

M3：Beta路径执行可用

交付：

  • 融合定位可用
  • 路线录制可用
  • 路线回放可用
  • 路径跟踪可演示

验收重点：固定路线可重复执行。

M4：障碍与安全可用

交付：

  • lidar 障碍检测可用
  • 安全停车可用
  • 边界约束可用
  • 降级逻辑可用

验收重点：不撞、不越界、不确定时停车。

M5：整机联调与交付

交付：

  • 端到端任务闭环
  • 标准演示路线
  • 源码
  • 文档
  • 部署说明
  • 测试记录
  • 交接记录

验收重点：可运行、可重复、可演示、可交接。

11. 核心验收标准

只接受以下结果：

  • 可运行
  • 可重复
  • 可演示
  • 可复现
  • 可交接

不接受：

  • 只演示一次
  • 只给视频
  • 只交二进制
  • 只讲算法
  • 不交源码
  • 不进 GitHub

12. 结算建议

建议按里程碑付款，不建议一次性大额预付。

付款应绑定：

  • 源码提交
  • 文档交付
  • 现场演示
  • 验收记录
  • 问题清单关闭

尾款应绑定最终交接。

13. 最终原则

后续 Gamma / Delta / Fleet / Service 能力，应在本阶段验收完成后另行定义范围、报价、里程碑和合同。
