3. 阶段类术语

3.1 Beta 阶段

Beta 阶段是基础任务执行阶段，不是最终无人驾驶阶段。

一句话：Beta = 能完成基础自动作业，但还需要人设定任务、监督和验收。

3.2 Alpha 阶段

Alpha 是更早阶段，目标是机器能跑、能遥控、能基本执行。

一句话：Alpha = 人主导，机器辅助。

3.3 Gamma 阶段

Gamma 是 Beta 后面的阶段，目标是有监督全自动。

通常包括：

  • 更稳定的 RTK + SLAM 融合
  • 全局路径规划
  • 区域覆盖算法
  • 多路径优化
  • 云端监控
  • 一人监控多机

一句话：Gamma = 机器人能更完整地自己规划和执行，但仍有人监督。

3.4 Delta 阶段

Delta 是更高级的无人值守阶段。

通常要求：

  • 全天候感知
  • 动态路径重规划
  • 无 GPS 兜底能力
  • 自动充电 / 自动任务调度
  • 异常自恢复
  • 长时间无人干预

一句话：Delta = 基本不用人盯着，机器人自己干活。

4. 导航与定位类术语

4.1 导航

导航不是单一功能，而是一整套能力。

导航需要回答：

一句话：导航 = 让机器人知道自己在哪，并按指定路线安全走过去。

4.2 定位

定位就是机器人判断自己当前的位置和方向。

一句话：定位 = 机器人知道自己在哪里、朝哪个方向。

4.3 RTK GNSS

RTK 是一种高精度卫星定位技术。

普通 GPS 可能误差几米，RTK 做得好可以达到厘米级。

SnowBot 使用 RTK 是为了让机器人在固定场地里知道自己的较精确位置。

优点：

  • 精度高
  • 适合开阔场地
  • 适合园区、停车场、道路

限制：

  • 需要基站或网络差分服务
  • 遮挡严重时效果变差
  • 建筑物、树木、金属环境可能影响信号

一句话：RTK = 高精度 GPS，用来让机器人知道自己在场地里的精确位置。

4.4 GNSS

GNSS 是卫星导航系统的统称。

GPS、北斗、Galileo、GLONASS 都属于 GNSS。

一句话：GNSS = 卫星定位总称，GPS 只是其中一种。

4.5 IMU

IMU 是惯性测量单元。

它通常测：

  • 加速度
  • 角速度
  • 姿态变化
  • 倾斜
  • 转弯
  • 震动

SnowBot 用 IMU 辅助判断机器人是否转弯、打滑、倾斜，以及在 RTK 信号变差时短时间辅助定位。

一句话：IMU = 机器人的平衡感和方向感。

4.6 里程计 / Odometry

里程计通过轮子或履带转动估算机器人走了多远、转了多少。

缺点是雪地打滑时会不准。

一句话：Odometry = 通过轮子或履带转动估算机器人走了多远。

4.7 融合定位

融合定位是把 RTK、IMU、里程计等信息合起来判断位置。

原因是：

  • RTK 精度高，但可能丢信号
  • IMU 反应快，但会漂移
  • 里程计连续，但打滑会错

一句话：融合定位 = 不相信单一传感器，把多个信息合起来判断位置。

4.8 位姿

位姿 = 位置 + 姿态。

地面机器人最常用的是：

一句话：位姿 = 机器人在哪里 + 朝哪个方向。

4.9 坐标系

坐标系是给机器人一个统一的位置参考标准。

例如把测试场地某一点设为 X=0、Y=0。

一句话：坐标系 = 给机器人一个统一的位置参考标准。

5. 路径与任务类术语

5.1 路径

路径是机器人要走的线。

一句话：路径 = 机器人要走的路线。

5.2 路线录制

路线录制是先由人遥控或引导机器人走一遍，把路线记录下来。

一句话：路线录制 = 人先带机器人走一遍，让它记住。

5.3 路线回放

路线回放是机器人按之前录制的路线再走一遍。

一句话：路线回放 = 机器人重复走之前记录好的路线。

5.4 Teach & Repeat

Teach & Repeat 是路线录制与回放的英文叫法。

Teach = 教它走一遍。

Repeat = 它自己重复走。

一句话：Teach & Repeat = 先教一遍，再自动重复。

5.5 路径规划

路径规划是机器人根据目标生成要走的路线。

Beta 阶段只要求：

  • 单路径
  • 简单曲线
  • 固定区域内基础路径

不要求复杂自动生成最优路线。

一句话：路径规划 = 机器人决定怎么从当前位置走到目标位置。

5.6 单路径规划

单路径规划只处理一条简单路线。

它不负责复杂区域覆盖，也不负责多台机器人协调。

一句话：单路径规划 = 只规划一条简单路线。

5.7 区域任务

区域任务是在一个被定义好的区域内作业。

例如：

  • 停车区
  • 人行道
  • 广场
  • 园区道路

一句话：区域任务 = 在指定范围内完成基础作业。

5.8 边界

边界是机器人不能越过的范围。

一句话：边界 = 机器人能活动的安全范围。

5.9 边界约束

边界约束是用软件规则限制机器人不越界。

如果接近边界，可以触发：

  • 减速
  • 停车
  • 报警
  • 降级

一句话：边界约束 = 软件上给机器人画红线。

5.10 路径跟踪

路径跟踪不是规划路线，而是让机器人沿着已经给定的路线走。

一句话：路径跟踪 = 让机器人尽量贴着既定路线走。

5.11 路径偏差

路径偏差是机器人实际走的位置和目标路线之间的误差。

一句话：路径偏差 = 实际走法和计划路线之间的差距。

6. 感知与避障类术语

6.1 感知

感知是机器人用传感器了解周围环境。

包括：

  • 前面有没有人
  • 前面有没有车
  • 有没有墙
  • 有没有雪堆
  • 有没有障碍物
  • 有没有越界风险

一句话：感知 = 机器人看见或探测周围环境。

6.2 激光雷达 / Lidar

激光雷达用激光扫描周围环境，生成距离信息。

它可以知道：

  • 前方多少米有障碍物
  • 障碍物大概形状
  • 障碍物距离
  • 空间边界

一句话：Lidar = 用激光测距离，帮助机器人发现障碍物和边界。

6.3 摄像头

摄像头用视觉图像识别环境。

Beta 阶段，摄像头可以作为辅助，但不建议作为唯一依赖。

一句话：摄像头 = 辅助看环境，但 Beta 阶段不能单独依赖它。

6.4 障碍物检测

障碍物检测是判断前方有没有东西挡路。

Beta 阶段重点是“发现并停下”，不是复杂识别每个物体是什么。

一句话：障碍物检测 = 发现前面有东西。

6.5 静态障碍物

静态障碍物是不动或几乎不动的障碍物。

例如：

  • 墙
  • 栏杆
  • 停着的车
  • 路桩
  • 垃圾桶

一句话：静态障碍物 = 不动或几乎不动的障碍。

6.6 动态障碍物

动态障碍物是会动的障碍物。

例如：

  • 行人
  • 行驶车辆
  • 其他设备

Beta 阶段不建议要求复杂动态障碍处理。当前重点是检测到风险后减速或停车。

一句话：动态障碍物 = 会动的人、车或物体。

6.7 安全区

安全区是机器人前方或周围设定的一块保护区域。

障碍物进入安全区后，机器人应触发：

  • 减速
  • 停车
  • 报警
  • 降级

一句话：安全区 = 机器人周围不能有危险物体靠近的保护范围。

6.8 安全停车

安全停车是机器人在发现风险时，能稳定、可控地停下来。

一句话：安全停车 = 发现风险后，机器人可靠停住。

6.9 简单绕行

简单绕行是在低速、低风险条件下绕过一个小障碍。

Beta 阶段不应强求复杂绕行。

一句话：简单绕行 = 低速绕开小障碍，但不是 Beta 阶段核心目标。

7. 控制与系统类术语

7.1 控制

控制是把“我要怎么走”变成底盘能执行的指令。

例如：

  • 前进
  • 转向
  • 减速
  • 停车

一句话：控制 = 把导航决策变成底盘动作。

7.2 底盘控制器

底盘控制器是负责驱动机器人运动的控制单元。

它接收上位机发来的命令，再控制电机、驱动器、制动、转向或差速。

一句话：底盘控制器 = 机器人运动执行的小脑。

7.3 IPC / 工控机

IPC 是工业电脑。

它通常运行：

  • Linux
  • ROS2
  • 导航程序
  • 感知程序
  • 任务状态机

一句话：IPC = 机器人本地大脑硬件。

7.4 上位机

上位机通常是机器人上运行智能软件的计算机，很多时候就是 IPC。

一句话：上位机 = 负责思考和协调的本地计算平台。

7.5 MCU

MCU 是微控制器，负责更底层、更实时的控制。

例如：

  • 电机控制
  • 急停
  • 限位
  • 继电器
  • 传感器底层采集
  • 安全链

一句话：MCU = 机器人底层实时控制的小脑。

7.6 ROS2

ROS2 是机器人软件常用的通信与开发框架。

它不是一个算法，而是让各模块协同运行的平台。

一句话：ROS2 = 机器人软件模块之间的通信和运行框架。

7.7 节点 / Node

节点是 ROS2 或机器人软件里的一个功能模块。

例如：

  • RTK 节点
  • IMU 节点
  • 定位节点
  • 路径规划节点
  • 控制节点
  • 安全监控节点

一句话：节点 = 一个独立运行的软件功能单元。

7.8 Topic

Topic 是 ROS2 里常见的信息发布通道。

一句话：Topic = 软件模块之间持续传消息的通道。

7.9 Service

Service 是 ROS2 里一问一答式的接口。

一句话：Service = 软件模块之间的一次性请求和响应。

7.10 Action

Action 是 ROS2 里用于长时间任务的接口。

一句话：Action = 用来执行长任务，并能反馈进度。

7.11 CAN

CAN 是工业设备和车辆中常用的通信总线。

一句话：CAN = 车辆和工业设备常用的可靠通信线。

7.12 UART / 串口

UART 是一种简单的串行通信方式。

一句话：UART = 简单设备之间常用的数据通信接口。

7.13 Ethernet / 以太网

以太网是高速稳定的数据通信方式。

一句话：以太网 = 高速稳定的数据通信方式。

8. 状态机与安全类术语

8.1 状态机

状态机是把机器人的运行过程拆成一个个状态，并规定每个状态之间怎么切换。

一句话：状态机 = 机器人工作流程的规则表。

8.2 idle

空闲状态，机器人等待指令。

一句话：idle = 等着，不干活。

8.3 ready

就绪状态，机器人准备好开始任务。

一句话：ready = 准备好了，可以开始。

8.4 manual assist

人工辅助状态，人在遥控或协助机器人完成操作。

一句话：manual assist = 人帮着机器人走。

8.5 route record

路线录制状态。

一句话：route record = 正在记路线。

8.6 route replay

路线回放状态。

一句话：route replay = 正在重复走已记录路线。

8.7 zone task active

区域任务执行状态。

一句话：zone task active = 正在指定区域里干活。

8.8 pause

暂停状态，机器人暂时停止任务，但任务还没结束。

一句话：pause = 暂停，后面还能继续。

8.9 obstacle stop

障碍停车状态。

一句话：obstacle stop = 看到障碍，停住。

8.10 degraded mode

降级模式是某些能力不可靠时，机器人降低功能级别继续安全运行，或者等待人工介入。

例如：

  • 降低速度
  • 停止自动任务
  • 只允许人工接管
  • 停在安全位置

一句话：degraded mode = 系统不完全正常时，降低能力保安全。

8.11 fault stop

故障停车。

一句话：fault stop = 出故障，停机等待处理。

8.12 task complete

任务完成状态。

一句话：task complete = 活干完了。

8.13 急停 / E-stop

急停是在紧急情况下强制停车。

急停不应完全依赖软件，最好有硬件安全链支持。

一句话：E-stop = 紧急情况下强制停车。

8.14 安全链

安全链是硬件和软件共同组成的安全回路。

例如：

  • 急停按钮
  • 防撞条
  • 安全继电器
  • MCU 安全输入
  • 电机使能切断

一句话：安全链 = 让机器人在危险时可靠停下来的安全系统。

9. 交付与验收类术语

9.1 源代码

源代码是软件原始代码。

一句话：源代码 = 软件的原始制作文件。

9.2 二进制文件

二进制文件是编译后的可执行程序。

一句话：二进制 = 做好的成品程序，不等于源码。

9.3 黑盒

黑盒是对方只给结果，不给内部逻辑、源码、参数和接口。

一句话：黑盒 = 能跑但看不懂、改不了、接不住。

9.4 部署

部署是把软件安装、配置、启动到机器人上的过程。

一句话：部署 = 把软件装到机器人上并跑起来。

9.5 参数

参数是软件运行时可调整的设置。

例如：

  • 最大速度
  • 停车距离
  • 安全区大小
  • 定位权重
  • 雷达检测阈值

一句话：参数 = 不改代码也能调的设置。

9.6 日志

日志是系统运行过程的记录。

一句话：日志 = 机器人运行时留下的证据。

9.7 rosbag / 回放数据

rosbag 是 ROS 里记录传感器和系统数据的文件。

一句话：rosbag = 把机器人运行过程录下来，后面可复盘。

9.8 测试报告

测试报告是证明系统是否真的完成的证据。

一句话：测试报告 = 验收的证据。

9.9 交接

交接是让 ARCBOS 的人能接得住系统。

交接包括：

  • 怎么部署
  • 怎么启动
  • 怎么调参数
  • 怎么看日志
  • 常见问题怎么处理

一句话：交接 = 让系统不再只掌握在外包方手里。

9.10 可复现

可复现是同样步骤、同样环境下，别人也能把结果跑出来。

一句话：可复现 = 不是只在外包方电脑上能跑。

10. 外包范围与付款类术语

10.1 In Scope

本次合同范围内，外包方必须交付。

一句话：In Scope = 这次必须交付。

10.2 Out of Scope

不在本次合同范围内，除非另行签约。

一句话：Out of Scope = 这次不做，别混进来。

10.3 里程碑 / Milestone

里程碑是阶段性交付节点。

例如：

  • M1 接口冻结
  • M2 定位跑通
  • M3 避障跑通
  • M4 整机集成
  • M5 交接完成

一句话：里程碑 = 分阶段验收和付款的节点。

10.4 Gate

Gate 是付款或决策关口。

一句话：Gate = 控付款、控进度的关口。

10.5 Blocker

Blocker 是阻断性问题。

例如：

  • 底盘接口不通
  • 定位完全不稳定
  • 程序无法部署
  • 安全停车不触发

一句话：Blocker = 不解决就走不下去的问题。

10.6 Bug 修复窗口

Bug 修复窗口是验收前后预留一段时间，让第三方修问题。

一句话：Bug 修复窗口 = 不是演示完就走，还要负责修问题。

10.7 尾款

尾款是最后一笔付款。

尾款应绑定：

  • 源代码
  • 文档
  • 部署说明
  • 测试报告
  • 交接培训

一句话：尾款 = 控制外包方认真收尾的最后抓手。

11. 最容易混淆的术语

11.1 路径规划 vs 路径跟踪

11.2 定位 vs 导航

11.3 避障 vs 安全停车

Beta 阶段优先安全停车，不优先复杂绕行。

11.4 Demo vs 可交付系统

11.5 源码交付 vs 系统交接

系统交接包括代码、文档、部署、参数、测试、培训。
