1）看提交是否一致

如果两串 SHA 一样，说明：

本地 main 和远端 main 在提交层面一致。

2）更直观看差异摘要

判断方法：

  • git diff --stat main origin/main 没输出

    = 文件内容一致

  • git log ... 看到两边指向同一个提交

    = 历史一致

  • git status 干净

    = 本地没有未提交内容

四、最精准检查

检查本地是否领先 / 落后远端

  • 左边 = 远端比本地多多少提交
  • 右边 = 本地比远端多多少提交

结果解释

  • 0 0

    完全一致

    本地比远端多 3 个提交，还没推上去

  • 2 0

    远端比本地多 2 个提交，本地落后

  • 2 3

    两边都改过，已分叉
