SnowBot Alpha/Beta 交付物与工程资料清单 v1.1

适用项目：SnowBot Alpha补齐 + Beta闭环

文件性质：合同附件 / 交付物清单 / 工程资料清单 / 披露清单

对应文件：附件一《SnowBot Alpha/Beta 导航与VCU控制技术基线包》、附件二《SnowBot Alpha/Beta 技术节点与验收付款条件》

1. 文件定位

本附件用于明确本项目各阶段应提交的源码、配置、协议、接口、测试数据、日志、硬件适配资料、第三方组件披露和工程交接资料。

附件一规定技术范围和最低技术要求；附件二规定技术节点、验收条件和付款触发条件；本附件规定各节点和最终交付时必须提交的工程资产和资料清单。

本附件不单独扩大乙方开发范围，但凡属于附件一和附件二约定范围内形成的项目成果、配置、参数、接口、测试记录和文档，均应按本附件要求交付。

2. 交付总原则

乙方正式交付应满足以下原则：

  • 可追踪：每项交付物应能对应版本、提交记录、测试记录或验收节点。
  • 可复现：甲方应能根据交付资料构建、部署、运行或复核测试过程。
  • 可接管：交付物不得依赖乙方个人电脑、乙方账号或乙方私有环境。
  • 可继续开发：源码、配置、协议、参数和接口资料应足以支持甲方或甲方委托第三方继续开发。
  • 可审查：第三方组件、硬件随附代码、授权限制、费用限制和使用限制应完整披露。

以下内容不得作为唯一交付：只交视频演示、截图、口头说明、二进制程序、乙方本地环境、控制器实物、传感器型号或单次演示结果。

3. GitHub与版本交付要求

乙方正式工程交付应进入甲方指定 GitHub 仓库和交付分支：

仓库：https://github.com/arcbos/snowbot-control
交付分支：control-work
编码：UTF-8
代码及注释：英文

源码、配置、协议、接口文档、测试脚本、测试报告和问题清单应提交至 control-work 分支，除非甲方书面确认其他交付方式。每个付款节点应对应明确的 commit、tag、release note 或等效版本记录。

4. 通用交付物清单

以下交付物适用于 P1-P5 各节点，具体提交时间以附件二节点要求为准：

通用交付物
├─ 源码
├─ 配置文件
├─ 参数备份
├─ 部署说明
├─ 构建说明
├─ 运行说明
├─ 接口文档
├─ 协议文档
├─ IO表
├─ 接线图
├─ 测试脚本或测试流程
├─ 测试数据
├─ 测试报告
├─ 运行日志、CAN日志、传感器日志或系统日志
├─ 问题清单
├─ 版本说明
└─ 交接说明

每项交付物应包含版本、日期、对应节点和负责人信息。测试报告应说明测试条件、测试对象、测试步骤、测试结果、异常情况和对应版本。

5. VCU控制系统交付物

VCU控制系统交付物至少包括：

VCU交付物
├─ VCU规格型号、接口和接入建议
├─ VCU配置文件
├─ VCU参数备份
├─ 项目定制VCU源码；如乙方开发
├─ VCU内置履带差速代码；如可随VCU交付
├─ CAN协议、CAN ID表或DBC文件
├─ 协议字段说明
├─ IO表
├─ 接线图
├─ 故障码表
├─ 底盘控制接口说明
├─ 作业电机控制接口说明
├─ 遥控控制接口说明
├─ 上位机控制接口说明
├─ 急停、软件停车、人工接管接口说明
├─ 安全停车逻辑说明
├─ 运行日志或CAN日志
├─ VCU测试报告
└─ VCU问题清单

VCU通用控制器自带的履带差速控制代码，如可随硬件交付，乙方应配合交付；如存在第三方授权、工具链、源码可见性或控制器厂商限制，乙方应在披露清单中说明。

6. 六电机控制交付物

六电机控制交付物应覆盖：左履带电机、右履带电机、除雪电机、抬升电机、旋转电机、俯仰电机。

最低交付物包括：六电机控制参数或配置文件、控制指令说明、启动/停止/状态反馈/故障反馈记录、桌面测试记录、装机联调测试记录、遥控器或上位机控制测试记录，以及与P1、P2节点对应的GitHub版本记录。

7. 导航与自主系统交付物

导航与自主系统交付物至少包括：

导航交付物
├─ 导航源码
├─ 导航配置文件
├─ 启动脚本或运行说明
├─ ROS2接口清单
├─ 传感器数据接入说明
├─ RTK/GPS、IMU、Odometry、Lidar、超声波数据说明；如适用
├─ 坐标系、TF和时序说明
├─ 定位与位姿接口说明
├─ A点/B点或路线目标配置说明
├─ 路线或目标点文件格式
├─ 任务状态机说明
├─ 障碍物检测、停车、有限绕行或安全停车说明
├─ 急停、软件停车、人工接管接口说明
├─ 导航日志格式
├─ 测试数据
├─ 测试报告
└─ 问题清单

A点至B点导航和障碍处理能力仅限附件一约定的 Alpha/Beta 工程样机阶段固定场地演示能力。交付资料应能说明测试场地、速度条件、传感器状态、障碍物条件、异常处理和对应版本。

8. 导航与感知硬件适配资料

导航系统所需激光雷达、超声波传感器、RTK/GPS模块、天线、IMU或其他导航/感知硬件，由乙方提供规格型号、接口要求、安装建议、软件接入要求和联调说明。

乙方应提交以下资料：推荐硬件名称、型号、规格、数据接口说明、供电要求、通信接口要求、安装位置/高度/方向/视场要求、坐标系和标定要求、软件接入说明、联调测试记录，以及相关授权或使用限制说明。

硬件取得、供应链、费用、质保、机械安装、电气供电和线束实施不属于乙方在本合同项下的交付责任，除非双方另有书面约定。

9. 测试资料与日志

乙方应按节点提交测试资料和日志：

测试资料与日志
├─ P1：六电机桌面测试记录、运行日志或CAN日志、问题清单
├─ P2：装机运动测试记录、急停/接管/停车测试记录、运行日志或CAN日志
├─ P3：目标点/路线文件、导航日志、传感器日志、P3测试报告
├─ P4：Alpha测试报告、Beta测试报告、系统联调测试报告、完整问题清单
└─ P5：质保期问题记录、缺陷关闭记录；如适用

日志应至少包含时间、事件类型、触发来源、系统状态、动作结果和对应版本。测试资料应能与GitHub提交记录、配置文件、运行日志和测试报告相互对应。

10. Release Package

每一付款节点对应的正式交付应形成 Release Package。P4节点应提交完整 Release Package。

完整 Release Package 至少包括：源码、VCU源码/配置/参数备份、VCU内置履带差速代码（如可随VCU交付）、导航源码与配置、部署说明、构建说明、运行说明、接口文档、协议文档、IO表、接线图、传感器规格型号/接口/安装建议、测试脚本或测试流程、测试数据、测试报告、运行日志、问题清单、版本说明、第三方组件和硬件随附能力披露清单、交接说明。

11. 第三方组件与硬件随附能力披露

乙方使用任何开源软件、第三方库、SDK、算法模型、工具链、控制器配置软件、云服务、外部API、硬件随附代码或闭源配置工具，应在交付时披露。

披露内容至少包括：名称、版本、来源、权利人或维护主体、许可证或授权方式、使用模块、是否修改源码、是否运行时依赖、是否涉及云服务或外部API、是否收费或订阅、是否影响甲方闭源商业化、复制、部署、修改、二次开发或委托第三方继续开发。

未披露、披露不完整或未经甲方确认的第三方组件，不得作为正式交付的一部分。

12. 背景技术披露

乙方如使用其签约前已合法拥有的背景技术，应在交付时披露。未披露的背景技术，不得作为限制甲方使用、复制、部署、修改、二次开发、商业化或委托第三方继续开发的理由。

背景技术披露至少包括：技术或模块名称、权利人、使用位置、授权范围、是否限制甲方商业使用、是否产生额外费用、是否影响知识产权转让和是否可由甲方继续开发。

14. 交接要求

最终交接时，乙方应向甲方说明代码目录结构、构建和部署方法、运行启动方法、主要配置文件、VCU参数和备份方式、CAN协议和IO表使用方式、导航传感器接入方式、日志查看和问题复盘方式、已知问题、后续建议、第三方组件和背景技术限制。

交接资料应足以支持甲方或甲方指定第三方继续维护和开发。

15. 与其他文件的关系

附件一规定技术范围、技术要求、工程边界和最低验收基线；附件二规定技术节点、验收条件和付款触发条件；本附件规定源码、配置、协议、资料、测试数据和交付物清单；主合同规定合同金额、付款安排、知识产权、保密、违约和争议解决。

如本附件与附件一或附件二存在不一致，应以附件一和附件二中更明确、且不扩大合同范围的要求为准。如本附件与主合同在知识产权、付款或违约责任方面存在不一致，以主合同为准。
